配置文件说明 {*必需：必需配置该标签（属性），内容可以为空；非必需：可以不配置该标签（属性）}

configure根标签

子标签 子标签说明 是否必需 子标签属性 子标签属性说明
basedir 生成代码文件的路径。如：d:\\code 是 没有
basePackage 生成代码的基础包名。如：com.test 是 没有
isDeleteTablePrefix 是否删除表名前缀。（true或false) 是 没有
columnIsCamel 是否表字段中下划线转成驼峰命名。（true或false) 是 没有
persistance 持久层框架。（mybatis,hibernate,other) 是 没有
common 公共类，形式：“包名=模板”多个英文逗号分隔，模板不需要后缀。主要是一些工具类，父类等,直接输出生成java文件，一般不需要。如：util=Page,commom.Dao=BaseDao 否 没有
db 连接数据库的配置，含有子标签，详情看db标签说明 是 没有
packageSetting 全局包设置，详情看packageSetting标签说明 是 没有
module 模块设置，可以配置多个module，详情看module标签说明 是 没有

db标签

子标签 子标签说明 是否必需 子标签属性 子标签属性说明
dbType 数据库。（mysql,oracle,sqlserver,informix,postgresql） 是 没有
dbName 数据库名 是 没有
user 数据库登陆用户 是 没有
pwd 数据库登陆密码 是 没有
driver 数据库驱动 是 没有
url 数据库url 是 没有

module标签（可配置多个）

子标签 子标签说明 是否必需 子标签属性 子标签属性说明
name 模块名。 是 没有
table 配置数据库表，一个模块下可以配置多个table 否 entityName,name, prefix,refType, refColumns,exclude entityName属性：配置自定义实体类名，非必需； name属性：表名，必需；prefix属性：表的前缀，去除前缀的时候需要，非必需； exclude属性：排除指定模板代码生成（entity,dao,service,controller,view,custom选择多个时，英文逗号分隔），非必需； refType属性：主从表的关系（OneToMany,OneToOne,ManyToOne,ManyToMany），从表必需，默认OneToMany。（主从关系说明看table标签） refColumns属性：主从表关联字段（主表字段=从表字段，多个英文逗号分隔），从表必需
entityPackage 自定义实体类包名。如：ai.entity。如未配置或标签内容为空字符，则该模块下不会去生成实体类，如下一样 否 没有
daoPackage 自定义dao接口包名,实现类默认在该包的impl下。如：ai.dao。 否 没有
servicePackage 自定义service接口包名,实现类默认在该包的impl下。如：ai.service。 否 没有
controllerPackage 自定义controller类包名。如：ai.controller。 否 没有
mapperPackage 自定义mapperXML包名。如：mapper。 否 没有
viewPackage 自定义视图层包名。如：ai.jsp。 否 tpl，type tpl属性：配置模板，只需模板名，不需要后缀，多个可以用英文逗号分隔，如view,list。必需属性; type属性：配置生成文件类型，即文件后缀，如jsp。非必需属性，默认‘jsp’
customPackage 自定义(自定义生成文件)包名。如：ai.custom。 否 tpl，type tpl属性：配置模板，只需模板名，不需要后缀，多个可以用英文逗号分隔，如indexJs。必需属性; type属性：配置生成文件类型，即文件后缀，如js。非必需属性，默认‘ctm’

table标签

子标签 子标签说明 是否必需 子标签属性 子标签属性说明
table table标签中可以包含多个table标签，table可以嵌套多层table。把这些表分为主表和从表，主表和从表是相对而言的。比如一个table在内部嵌套了一个table， 则外部的是主表，内部的是从表，如果这个从表内部还有table，则相对于它内部的表来说，它是主表了。需要注意的是主表一个，从表可以多个，不要重复配置表，会打乱表之间关系。理论上可以嵌套无限层table。 从表必需设置refType属性（主表对从表的对应关系），默认OneToMany（一对多）；必需设置refColumns属性， 主从关联字段（主表字段=从表字段，多个英文逗号分隔） 否 参照module标签说明 参照module标签说明
